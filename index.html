Sección 1
Conceptos clave
Todo lo que necesitas entender antes de tocar el teclado.
🖧

Dirección IP

Identificador único de cada dispositivo en la red. Tiene 32 bits divididos en 4 octetos. La red es la "colonia" y el host es la "casa".

192.168.1.10
📏

Máscara de subred

Define qué parte de la IP es red y qué parte es host. /24 = 255.255.255.0 = 254 hosts disponibles en la misma red.

255.255.255.0
🚪

Gateway (puerta de enlace)

El router que conecta la red local con el exterior. Toda PC necesita saber su dirección para salir a internet.

192.168.1.1
🏢

VLAN

Red lógica dentro de un switch físico. Separa el tráfico entre departamentos sin necesidad de hardware extra. Un piso = una VLAN.

Virtual LAN
🔌

Puerto Access

Conecta un dispositivo final (PC) a UNA sola VLAN. Es el tipo de puerto que usas para las computadoras de la red.

switchport access
🔐

Enable Secret

Contraseña cifrada con MD5 que protege el modo privilegiado (#) del router y switch. Siempre usar en lugar de enable password.

MD5 cifrado
🛡

Port-Security

Limita qué dispositivos pueden conectarse a un puerto por su dirección MAC. Si entra una MAC no autorizada, el puerto se apaga.

MAC address
🔒

SSH

Protocolo de administración remota cifrada. Reemplaza a Telnet (que envía contraseñas en texto plano). Requiere clave RSA.

Puerto 22
📡

Ping

Envía paquetes ICMP para verificar si hay comunicación entre dos IPs. 0% pérdida = conexión OK. 100% pérdida = sin ruta o VLAN mal configurada.

ICMP
🗺

Tracert

Muestra todos los saltos (routers) que recorre un paquete hasta su destino. El primer salto siempre debe ser tu gateway.

tracert 8.8.8.8
🔍

DNS (nslookup)

Traduce nombres de dominio a IPs. Si fallas un ping por nombre pero sí por IP, el problema es el DNS, no la red.

nslookup
💾

Running vs Startup Config

Running-config = activa en RAM (se pierde al reiniciar). Startup-config = guardada en flash. Siempre guardar antes de cerrar Packet Tracer.

copy run start
Sección 2
Comandos usados hasta ahora
Solo los comandos que hemos practicado en clase. Nada más.
💻 Diagnóstico desde la PC (Command Prompt / Windows)
ipconfig /all Ver IP, máscara, gateway y MAC del equipo Primer comando del técnico
ping [IP] Verificar conectividad con otro equipo Comprobar comunicación
tracert 8.8.8.8 Ver la ruta hasta internet (salto a salto) Localizar dónde falla
nslookup [dominio] Verificar que el DNS traduce nombres Problema de nombres
ssh -l admin [IP] Conectarse al router por SSH cifrado Administración remota
⌨ Modos del CLI (Switch / Router Cisco)
Switch> enable Entrar al modo privilegiado # Siempre primero
Switch# configure terminal Entrar al modo configuración global (config)# Para modificar config
(config)# end Salir directo al modo privilegiado Salida rápida
(config)# exit Subir un nivel en el modo actual Paso a paso
🏢 VLANs en el switch
vlan 10 Crear VLAN con ID 10. Prompt cambia a (config-vlan)# Crear segmento
name Administracion Poner nombre descriptivo a la VLAN Dentro de config-vlan
show vlan brief Ver todas las VLANs y sus puertos asignados Verificar y diagnosticar
interface fa0/1 Seleccionar puerto. Prompt: (config-if)# Configurar puerto
interface range fa0/1-3 Seleccionar varios puertos a la vez. Prompt: (config-if-range)# Atajo para varios
switchport mode access Definir el puerto como acceso (una sola VLAN). Obligatorio antes del siguiente. Siempre antes de access vlan
switchport access vlan 10 Asignar el puerto a VLAN 10 Asignar segmento
🔐 Seguridad (contraseñas + port-security + SSH)
enable secret [clave] Contraseña cifrada MD5 para modo privilegiado. Nunca usar enable password. Lo más importante
service password-encryption Cifra todas las contraseñas en texto plano del running-config Después de cada clave
line console 0 Entrar a config de la consola física. Agregar: password + login Acceso físico
line vty 0 4 Entrar a config de acceso remoto (Telnet/SSH). Agregar: password + login Acceso remoto
switchport port-security Activar port-security en el puerto (debe estar en modo access primero) Control de MAC
switchport port-security
mac-address sticky		 Aprender automáticamente la MAC del dispositivo conectado Más fácil en clase
switchport port-security
violation shutdown		 Apagar el puerto si entra una MAC no autorizada Respuesta ante intruso
ip domain-name [nombre] Nombre de dominio obligatorio para generar clave RSA de SSH Paso 1 de SSH
crypto key generate rsa
modulus 1024		 Generar par de claves RSA. En Packet Tracer usar 1024. Paso 3 de SSH
ip ssh version 2 Forzar SSH versión 2 (más segura) Paso 4 de SSH
transport input ssh Bloquear Telnet y permitir solo SSH en las líneas VTY En line vty 0 4
✅ Verificación y guardado
show running-config Ver toda la configuración activa. Aquí ves contraseñas, VLANs, interfaces. Auditar configuración
show ip interface brief Ver estado de todas las interfaces. Busca up/up. Estado de puertos
show port-security interface fa0/1 Ver estado de port-security en un puerto específico Verificar port-sec
show ip ssh Verificar que SSH está habilitado y su versión Verificar SSH
copy running-config startup-config Guardar la configuración. Sin esto se pierde al reiniciar. SIEMPRE al final

Escenario empresarial

Logística Express S.A. — Red segura con VLANs

La empresa acaba de mudarse. Te contratan para construir su red desde cero: 3 departamentos separados, contraseñas en todos los equipos y acceso seguro por SSH.

01
Construir la topología
▼
Pregunta
¿Qué componentes necesitas agregar al canvas y cómo los conectas?
Piensa antes de arrastrar: ¿qué switch? ¿qué cable entre PC y switch?

Lo que debes hacer:

  • Agregar 1 switch Catalyst 2960 al centro del canvas
  • Agregar 7 PCs: 3 para Admin (izquierda), 2 para Operaciones (derecha arriba), 2 para Gerencia (derecha abajo)
  • Conectar cada PC al switch con cable Copper Straight-Through (el recto, no el cruzado)
  • Esperar a que los puertos pasen de naranja a verde antes de continuar
Si algún puerto queda en amarillo o naranja permanente, el cable está mal. Bórralo y vuelve a conectar con el cable correcto.
02
Crear las 3 VLANs en el switch
▼
Pregunta
¿Qué secuencia de comandos usas para crear una VLAN con nombre en el switch?
Recuerda: primero el modo, luego el ID, luego el nombre.
-- Entrar al CLI del switch -- Switch> enable Switch# configure terminal -- Crear VLAN 10 -- Switch(config)# vlan 10 Switch(config-vlan)# name Administracion Switch(config-vlan)# exit -- Crear VLAN 20 y VLAN 30 de la misma forma -- Switch(config)# vlan 20 Switch(config-vlan)# name Operaciones Switch(config-vlan)# exit Switch(config)# vlan 30 Switch(config-vlan)# name Gerencia Switch(config-vlan)# exit
Verifica
Ejecuta show vlan brief — ¿aparecen las 3 VLANs? ¿Tienen puertos asignados todavía?
Normal en este punto: las VLANs aparecen pero sin puertos. Todos siguen en VLAN 1.
Si el prompt NO cambia a (config-vlan)# al escribir vlan 10, estás en un switch no administrado. Usa el modelo 2960, no el genérico.
03
Asignar puertos a cada VLAN
▼
Pregunta
¿Cuáles son los DOS comandos obligatorios para asignar un puerto a una VLAN? ¿Qué pasa si usas solo uno?
-- Asignar Fa0/1, Fa0/2, Fa0/3 a VLAN 10 (usando range) -- Switch(config)# interface range fa0/1-3 Switch(config-if-range)# switchport mode access Switch(config-if-range)# switchport access vlan 10 Switch(config-if-range)# exit -- Fa0/4, Fa0/5 a VLAN 20 -- Switch(config)# interface range fa0/4-5 Switch(config-if-range)# switchport mode access Switch(config-if-range)# switchport access vlan 20 Switch(config-if-range)# exit -- Fa0/6, Fa0/7 a VLAN 30 -- Switch(config)# interface range fa0/6-7 Switch(config-if-range)# switchport mode access Switch(config-if-range)# switchport access vlan 30 Switch(config-if-range)# exit
Verifica
Ejecuta show vlan brief — ¿aparecen los puertos correctos en cada VLAN?
VLAN 10: Fa0/1-3 · VLAN 20: Fa0/4-5 · VLAN 30: Fa0/6-7
04
Configurar IPs en las PCs
▼
Pregunta
¿Dónde se configura la IP en Packet Tracer y qué diferencia hay entre DHCP y Static?
En esta práctica usamos Static — ¿por qué crees que preferimos asignar IPs a mano?

Ruta en Packet Tracer:

Doble clic en cada PC → pestaña Desktop → IP Configuration → seleccionar Static → llenar IP, Subnet Mask y Default Gateway.

Usa la tabla de referencia de arriba. Repite para las 7 PCs.

El campo Gateway puedes dejarlo vacío por ahora — no tenemos router todavía. Se configura cuando agregues el router en la siguiente actividad.
05
Probar conectividad con ping
▼
Pregunta 1
¿Qué resultado esperas al hacer ping entre dos PCs de la MISMA VLAN?
Desde PC-Admin1 → ping 10.10.10.2
PC-Admin1> ping 10.10.10.2 Reply from 10.10.10.2: bytes=32 time=1ms TTL=128 Reply from 10.10.10.2: bytes=32 time=1ms TTL=128 Sent=4, Received=4, Lost=0 (0%) ← CORRECTO
Pregunta 2 — IMPORTANTE
¿Qué resultado esperas al hacer ping entre dos PCs de VLANs DIFERENTES?
Desde PC-Admin1 → ping 10.20.20.1 (Operaciones)
PC-Admin1> ping 10.20.20.1 Request timeout for icmp_seq 0 Request timeout for icmp_seq 1 Sent=4, Received=0, Lost=4 (100%) ← TAMBIÉN CORRECTO
El fallo entre VLANs distintas ES el objetivo. Significa que la segmentación funciona. Si responde, hay un error de configuración.
06
Agregar contraseñas al switch
▼
Pregunta
¿Por qué usamos enable secret y no enable password?
Pista: ejecuta show running-config después de configurar cada uno y observa la diferencia.
Switch(config)# enable secret Cisco2024 Switch(config)# service password-encryption -- Proteger consola física -- Switch(config)# line console 0 Switch(config-line)# password ConsolaS1 Switch(config-line)# login Switch(config-line)# exit -- Proteger acceso remoto -- Switch(config)# line vty 0 4 Switch(config-line)# password VTY2024 Switch(config-line)# login Switch(config-line)# exit
El comando login es obligatorio. Sin él la contraseña existe pero nunca se solicita.
07
Activar Port-Security en los puertos
▼
Pregunta
¿Qué hace el parámetro sticky y por qué es útil en clase?
¿Cómo sabría el switch cuál es la MAC autorizada sin que la escribas a mano?
Switch(config)# interface range fa0/1-7 Switch(config-if-range)# switchport port-security Switch(config-if-range)# switchport port-security maximum 1 Switch(config-if-range)# switchport port-security violation shutdown Switch(config-if-range)# switchport port-security mac-address sticky Switch(config-if-range)# exit
Prueba visual
Desconecta PC-Admin1 de Fa0/1 y conecta una PC nueva al mismo puerto. ¿Qué le pasa al puerto?
Si el puerto se pone rojo/naranja: port-security funciona perfectamente.
Para reactivar un puerto en err-disable: interface fa0/1 → shutdown → no shutdown
08
Configurar SSH en el router
▼
Pregunta
¿Cuál es la diferencia entre Telnet y SSH? ¿Por qué Telnet es un problema de seguridad?
Piensa: si alguien en la red captura el tráfico de Telnet, ¿qué ve?
-- En el router 1941: 4 pasos para SSH -- Router(config)# ip domain-name logistica.local Router(config)# username admin privilege 15 secret Admin1 Router(config)# crypto key generate rsa modulus 1024 Router(config)# ip ssh version 2 Router(config)# line vty 0 4 Router(config-line)# transport input ssh Router(config-line)# login local Router(config-line)# exit
Verifica
Desde una PC: ssh -l admin 192.168.1.1 — ¿el prompt del router aparece en la PC?
Si SSH dice "disabled": falta el ip domain-name o la clave RSA. En Packet Tracer usa siempre modulus 1024, no 2048.
09
Guardar la configuración
▼
Pregunta final
¿Qué diferencia hay entre running-config y startup-config?
¿Qué pasa si cierras Packet Tracer sin ejecutar el siguiente comando?
Switch# copy running-config startup-config Destination filename [startup-config]? <- presiona Enter Building configuration... [OK] -- Atajo equivalente: -- Switch# wr
Repite este comando en el router también. Guarda siempre los dos dispositivos antes de cerrar Packet Tracer.

  • ping dentro de la misma VLAN responde (0% pérdida)
  • ping entre VLANs distintas falla (100% pérdida)
  • show running-config muestra contraseñas cifradas
  • Puerto Fa0/1 entra en err-disable al conectar PC intrusa
  • show ip ssh muestra SSH Enabled version 2.0
  • ssh -l admin [IP] entra al router desde una PC
