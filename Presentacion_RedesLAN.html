Conceptos clave

Lo que debes entender antes de tocar el teclado.

🌐
Dirección IP
Identificador único de cada dispositivo. 32 bits en 4 octetos. La red es la colonia y el host es la casa. Dos equipos en la misma red deben tener los mismos primeros octetos.
192.168.1.10 / 24
📏
Máscara de subred
Define qué parte de la IP es red y qué parte es host. /24 = 255.255.255.0 = 254 hosts usables. Si dos IPs están en subredes distintas, no se pueden comunicar sin router.
255.255.255.0 = /24
🚪
Gateway
Puerta de salida al exterior. Es la IP del router en la misma subred. Sin gateway correcto la PC puede hablar con la LAN pero no puede salir a internet.
192.168.1.1
🏢
VLAN
Red lógica dentro de un switch físico. Un switch = varios departamentos aislados. Sin VLAN todos comparten el mismo tráfico. Con VLAN cada departamento es su propio "piso" del edificio.
VLAN 10 / 20 / 30
🔌
Puerto Access
Conecta un dispositivo final a UNA sola VLAN. Necesita dos comandos: mode access primero, luego access vlan. Si falta uno, el puerto queda en VLAN 1 sin importar lo que escribas.
switchport mode access
🔐
Enable Secret
Protege el modo privilegiado (#) con cifrado MD5. Nunca usar enable password — guarda la clave en texto plano. Con service password-encryption se cifran las demás claves.
enable secret Cisco2024
🛡
Port-Security
Solo la MAC autorizada puede usar el puerto. sticky aprende automáticamente la MAC conectada. violation shutdown apaga el puerto ante un intruso.
port-security + sticky
🔒
SSH vs Telnet
Telnet: texto plano — cualquiera con Wireshark ve tu contraseña. SSH: cifrado RSA — tráfico ilegible. Necesita: domain-name → usuario → clave RSA → ip ssh version 2.
ssh -l admin [IP]
📡
Herramientas de diagnóstico
ipconfig /all — ver IP/MAC del equipo. ping — verificar conectividad. tracert — ver todos los saltos hasta el destino. nslookup — probar resolución DNS.
ping · tracert · nslookup

Reglas de oro del técnico

Memoriza estas 8 reglas. Responderlas en la evaluación marca la diferencia.

01
enable secret, nunca enable password
enable password guarda la clave en texto plano. enable secret usa MD5. Si ves la contraseña legible en show running-config, el equipo está en riesgo.
enable secret Cisco2024
02
Dos comandos para asignar VLAN a un puerto
switchport mode access PRIMERO, luego switchport access vlan X. Si falta el primero, el puerto puede ignorar la asignación de VLAN.
mode access → access vlan 10
03
show vlan brief es el primer diagnóstico
Antes de tocar cualquier configuración de VLAN, ejecuta show vlan brief. Si el puerto no aparece en la VLAN correcta, ahí empieza el problema.
show vlan brief
04
El fallo entre VLANs distintas es correcto
Si ping entre VLAN 10 y VLAN 20 falla con 100% pérdida, la segmentación funciona perfectamente. Si responde, hay un error de configuración.
Request timeout ← CORRECTO
05
SSH necesita domain-name primero
Sin ip domain-name, el comando crypto key generate rsa no puede ejecutarse. El nombre de dominio forma parte del identificador de la clave RSA.
ip domain-name → crypto key
06
Puerto err-disable no se arregla solo
Cuando port-security detecta un intruso, apaga el puerto en err-disable. Para reactivarlo: shutdown → no shutdown. Primero reconecta el dispositivo autorizado.
shutdown → no shutdown
07
login es obligatorio en line console y VTY
Si configuras la contraseña pero olvidas el comando login, la contraseña existe pero nunca se solicita al conectarse. El acceso queda abierto.
password X → login
08
Guarda siempre antes de cerrar
running-config vive en RAM y se pierde al reiniciar. copy running-config startup-config (o simplemente wr) copia todo a la memoria flash permanente.
copy run start / wr
